A Nix flake that builds a Neovim configuration via [nixvim](https://github.com/nix-community/nixvim). All Neovim config lives as Nix expressions; there are no Lua/Vimscript dotfiles outside the flake.

## Build & Verify

| Command | What it does |
|---------|-------------|
| `nix run` or `just run` | Build and launch the configured Neovim |
| `nix flake check --show-trace` or `just check` | Run nixvim test derivations for the package |
| `nix flake update` or `just update` | Update flake.lock |
| `just --list` | Show all justfile recipes |

## Architecture

- **Entrypoint**: `flake.nix` uses `flake-parts` + `import-tree` to auto-import every `.nix` under `modules/`.
- **Module convention**: Each file exports `flake.modules.nixvim.<name>`. These are collected in `modules/packages/neovim.nix` via `lib.attrValues config.flake.modules.nixvim` and fed to `nixvim.makeNixvimWithModule`.
- **Outputs**:
  - `packages.default` — the built Neovim package.
  - `legacyPackages.nixvimConfigurations.default` — the raw nixvim configuration.
  - `checks` — one test derivation per package, generated by `nixvim.lib.<system>.check.mkTestDerivationFromNvim`.
- **Supported systems**: `x86_64-linux`, `aarch64-darwin`.

## Directory Layout

```
modules/
  core/         # options, keymaps, autocmds, LSP, diagnostics, completion
  languages/    # per-language LSP, formatter, linter configs
  plugins/      # plugin-specific configs (telescope, git, UI, etc.)
  packages/     # flake outputs: neovim build, checks, nixpkgs overlay wiring
```

## Key Configuration Facts

- **Leader key**: `<Space>` (`mapleader = " "`).
- **Aliases**: `vi` and `vim` both point to this Neovim.
- **Indent**: 2 spaces (`tabstop = 2`, `shiftwidth = 2`, `expandtab = true`).
- **Clipboard**: `unnamedplus` (system clipboard).
- **Disabled providers**: ruby, perl, python2.
- **File manager**: Yazi (`<leader>e` / `<leader>E`). nvim-tree is present but commented out.
- **Lazy loading**: `lz-n` is enabled; many plugins use `lazyLoad.settings`.

## Formatting & Linting

- **Formatter**: `conform-nvim` with `format_on_save` (synchronous, 200ms timeout, falls back to LSP). Slow filetypes are tracked per-buffer and switch to async after the first timeout.
- **Disable autoformat per buffer**: `vim.b[bufnr].disable_autoformat = true`; globally: `vim.g.disable_autoformat = true`.
- **Linter**: `nvim-lint` triggered on `BufReadPost`, `BufWritePost`, `InsertLeave`.
- **Nix formatting**: `nixfmt` (not `alejandra` or `nixpkgs-fmt`).

## Testing

- Uses `neotest` with the Go adapter (`neotest-go`). No keymaps are bound for it yet.
- Checks are nixvim-level smoke tests, not language-specific test suites.

## Adding a Language

Create a file under `modules/languages/<name>.nix` that exports `flake.modules.nixvim.languages` and sets:
- `plugins.lsp.servers.<server>.enable = true;`
- `plugins.conform-nvim.settings.formatters_by_ft` and `formatters.<name>.command = lib.getExe pkgs.<pkg>;`
- `plugins.lint.lintersByFt` and `linters.<name>.cmd = lib.getExe pkgs.<pkg>;` if applicable.

No other registration is needed; `import-tree` picks it up automatically.
